# Kivy FFmpeg Video Compressor

A simple desktop app built with **Python** and **Kivy** that lets you:

* select a video file from your computer
* configure compression settings
* run **ffmpeg** directly from the command line
* save the compressed output video
* inspect the generated command and ffmpeg logs

This project is useful for compressing large lecture/class videos with a practical interface instead of writing commands manually every time.

---

## Features

* Video file picker
* Output file naming helper
* H.265 (`libx265`) and H.264 (`libx264`) support
* Configurable preset, CRF, and audio bitrate
* Optional resize to 720p
* Direct execution of `ffmpeg`
* Live command preview
* Log output window
* Stop running compression process

## 1. Install Python dependencies

Open **Command Prompt** or **PowerShell** inside the project folder and run:

```bash
pip install kivy
```

You can verify installation with:

```bash
python -c "import kivy; print(kivy.__version__)"
```

---

## 2. Install ffmpeg on Windows

### Option A — manual installation from official builds

1. Download a Windows build of **ffmpeg** from the [ffmpeg windows builds](https://www.ffmpeg.org/download.html#build-windows).
2. Open the download page and choose a Windows build provider.
3. Download a release build ZIP archive.
4. Extract the ZIP file to a folder such as:

```text
C:\ffmpeg
```

After extraction, you should have a path similar to:

```text
C:\ffmpeg\bin\ffmpeg.exe
```

### Add ffmpeg to PATH

1. Press **Start** and search for:

```text
Environment Variables
```

2. Open **Edit the system environment variables**.
3. Click **Environment Variables...**
4. Under **User variables** or **System variables**, select **Path**.
5. Click **Edit**.
6. Click **New** and add:

```text
C:\ffmpeg\bin
```

7. Confirm all windows with **OK**.
8. Close and reopen Command Prompt or PowerShell.

### Verify ffmpeg installation

Run:

```bash
ffmpeg -version
```

If installed correctly, you should see version information instead of an error.

## How to use

1. Click **Browse** and select a video.
2. Check the suggested output file name.
3. Choose compression settings:

   * **Codec**: `libx265` for stronger compression, `libx264` for better compatibility
   * **Preset**: `slow` is a good default
   * **CRF**: lower = better quality, higher = smaller file
   * **Audio kbps**: for spoken lectures, `64` to `96` is usually enough
4. Optionally enable **Resize to 720p**.
5. Click **Build command** to preview the ffmpeg command.
6. Click **Compress video**.
7. Check the log window for ffmpeg output.

---

## Suggested settings for class videos

### Balanced compression

* Codec: `libx265`
* Preset: `slow`
* CRF: `28`
* Audio: `96k`

### Stronger compression

* Codec: `libx265`
* Preset: `slow`
* CRF: `30`
* Audio: `64k`

### Better compatibility

* Codec: `libx264`
* Preset: `slow`
* CRF: `23`
* Audio: `96k`

## Example ffmpeg command generated by the app

```bash
ffmpeg -n -i input.mp4 -c:v libx265 -preset slow -crf 28 -c:a aac -b:a 96k output_compressed.mp4
```

Example with 720p resize:

```bash
ffmpeg -n -i input.mp4 -vf "scale=-2:720" -c:v libx265 -preset slow -crf 28 -c:a aac -b:a 96k output_720p.mp4
```

## Troubleshooting

### `ffmpeg not found in PATH`

This means Windows cannot find `ffmpeg.exe`.

Check that:

* `ffmpeg` is installed
* `C:\ffmpeg\bin` is in your `Path`
* you reopened the terminal after editing environment variables

Test again with:

```bash
ffmpeg -version
```

### `python is not recognized`

Python is either not installed or not added to PATH.

Try:

```bash
py --version
```

If `py` works, use:

```bash
py kivy_ffmpeg_video_compressor.py
```

### Kivy installation issues

Update pip first:

```bash
python -m pip install --upgrade pip
```

Then reinstall Kivy:

```bash
pip install --upgrade kivy
```

### Output file already exists

Enable **Overwrite output** in the app, or change the output filename.

## Notes

* `libx265` usually provides much smaller files than `libx264`, but some older devices may have weaker compatibility.
* Lecture videos with slides and speech can usually tolerate relatively strong compression.
* If the source video is Full HD or larger, resizing to 720p can reduce size significantly.
